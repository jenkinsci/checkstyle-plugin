Jenkins kann Checkstyle Reports einlesen und darstellen. Wenn dies aktiviert wird und der Build das Maven Goal checkstyle:checkstyle ausführt, dann analysiert Jenkins nach jedem Build die erzeugten Checkstyle Dateien und stellt deren Informationen in verschiedenen Ansichten dar: Trend Anzeige, Projekt Statistik und Package Statistik. Zudem zeigt Jenkins die jeweiligen Warnungen direkt in der betroffen Datei an.

Das Plug-in wird nicht aufgerufen, wenn der Build fehlgeschlagen ist, sondern nur für stabile bzw. instabile Builds (mit fehlgeschlagenen Tests).
